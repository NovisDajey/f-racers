General
For the best performance and results please use R-Tune Vehicle Physics with the minimum version of Unreal Engine 5.2. Older versions (5.0, 5.1) are supported but do not perform as well as newer versions.

High Speed Replication
If your server is able to handle high update rates, this replication setup can be used.


  

In order to enable network replication, simply set the replication method to 'Full'. Replication is fully integrated and requires no component setup. Ensure that ‘Replicate Movement’ is set to false in the default replication settings.   
If this setting is enabled the custom movement will not work. 


The replicated movement is not limited to the R-Tune implementation. The vehicle system is designed such that 3rd party plugins like Smooth Sync can also be used and work seamlessly. If a 3rd party or custom movement solution is used the replication method should be set to either ‘Data Only’ or ‘None’.


For replication to work correctly ensure that ‘Tick Physics Async’ is enabled in Project Settings -> Physics -> Frame Rate. Using values equal to or slightly less than 0.008333 (1/120) is advised.

Interior Animation Component (2.0)
Animating components like the steering wheel, RPM needle and speed needle can be done by using the ‘RTune Interior’ component. No extra setup is required in blueprint. Once added, R-Tune will automatically handle the updates:


  

















This is a mesh based component with the animation/component type specifiable:
  



  

The animation type is used to specify what data will be used to update the component. To control how the component functions the animation properties can be changed:
* Input Range: The range for the data being used to drive animation(RPM, Speed, Steering). 
   * E.g for RPM the minimum (X) will be the idle RPM, and the maximum (Y) will be the max rpm. 
   * For Steering the minimum (X) will be -1, and maximum 1 (Y). 
   * For Speed the minimum (X) will be 0, and maximum (Y) will be the top speed 
* Output range: The range that the input will be mapped to.
* Rotation Axis: How the component rotates (Roll, Pitch, Yaw).
* Use Smoothing: Whether to increase smoothness of animation. This is at the cost of accuracy and response.
* Smoothing Speed: How quickly to transition to the next state. 
* Invert: To reverse the direction of the animation, enable this.

AI (Experimental) (2.0)
To use AI functionality, first add the ‘RTune AI’ component.
  

  

The AI system works by following a spline component. The spline component will be a component of an actor. To get started use the ‘BP_Spline’ actor:
  















Once the ‘BP_Spline’ actor has been placed in the world this needs to be set in the vehicle. The R-Tune AI vehicle already has this setup, so you can simply select the spine actor in the outliner:
  




To modify how the vehicle behaves, the PID settings will be changed:
  



* Kp: Proportional gain coefficient. This controls how quickly the system responds. Setting this too high can cause instability and overshooting.
* Kd: Derivative coefficient. This is useful for damping the movement and increasing smoothness.
* Kd2: Second derivative coefficient. This can be used to damp the system as a whole. This is more useful than 'Kd' for smoothing
* Ki: Integral gain coefficient. This is useful in helping correct steady state offsets.
* Max Integral: Maximum buildup for integral accumulation (Ki)
* Output Max: Maximum steering output. This is effectively a clamp
* Min Delta Time: This is the minimum delta time that integral and derivative computations will take place.

Setup Modifications
Center of Mass
The center of mass of the vehicle’s collision mesh will have an effect on the vehicle’s handling. This means that for blueprints with identical settings, but different meshes for the root component (collision mesh), will have slightly different handling behaviour. This is due to the shape's inertia, and how forces interact with that specific shape. To eliminate this effect there are a few options: 


1. Center of Mass Offset: In the core mesh settings, the center of mass can be offset in the X-axis direction (first entry):
  

For more naturally drifty behaviour the offset should be negative. To correct for naturally drifty behaviour this can be set to a positive value. A good starting value to use is 20, this is based on the size of the mesh so higher or lower values are also valid. The Z-axis (last entry) can also be used to improve stability. If the vehicle has a tendency to roll over this offset can be set to a negative value. If the vehicle is leaning in the steering direction it can be set to a positive value.
  

The center of mass is the central location of the collision mesh (green outline). This center of mass offset changes its location in this shape.




2. Separate collision and rendering meshes: If there are multiple different vehicles used and the handling characteristics are to be shared, this method should be used:
  

Another static mesh component (Rendering Mesh) is used with all collision and physics disabled (simulate physics and mass disabled). The collision preset should be set to ‘Custom’ with all collision responses set to ‘Ignore’:
  



  

The core mesh uses a static mesh, with collision setup, except the visibility is disabled so that it does not render. This will allow for a different (physics) collision mesh to be used that is independent of the visible rendering mesh.

FAQ
Car jumps while still
If the vehicle seems to be jumping position after stopping, the idle lock stiffness can be decreased. Alternatively, it can be disabled completely:


  

Errors when opening project
Older versions of the plugin will not work with newer demo projects. Please make sure that the latest official version of the plugin is installed. If things are broken - this is the issue.
Bugs
Vehicle not moving/stuck in Multiplayer:
* If the vehicle is not moving, even though input is being registered. Disable the idle lock completely.


Vehicle’s rotation desync when replicated:
* In the project settings set the Angle Lerp to 0.05, or some other low value
* This is only for the optimal setup with replication set to ‘Data Only’


Scaling actor breaks physics:
* Vehicles cannot be scaled. If vehicles need to be larger, a larger mesh should be used with the actor scale set to 1.
